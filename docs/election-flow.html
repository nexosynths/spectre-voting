Spectre Election Flow

End-to-end anonymous voting with threshold committee decryption. Every step happens on-chain — no trust required.

Setup
1
Admin

Create Election

Set the question, voting options, signup gate, and assign committee members by wallet address.

2
Committee

Register Keys

Each committee member generates a keypair and registers their public key on-chain.

3
Admin

Finalize Committee

Run the dealer ceremony to generate the election encryption key and distribute encrypted shares.

4
Voters / Admin

Register Voters

How voters register depends on the signup gate the admin chose — open link, invite code, allowlist identifier, or admin-registered. This must happen before voting opens.

5
Admin

Close Signup & Open Voting

Lock the voter list and open the voting period. No more registrations after this point.

Voting
6
Voters

Vote

Pick an option and submit. Behind the scenes, a ZK proof anonymizes your identity and your vote is encrypted.

7
Admin

Close Voting

End the voting period. No more votes accepted after this point.

Results
8
Committee

Decrypt Shares

Committee members submit their decrypted key shares on-chain. Only the threshold is needed (e.g., 2 of 3).

9
Automatic

Tally Votes

Once enough shares are submitted, votes are automatically decrypted and counted in the browser.

10
Admin

Commit Results

Publish the final vote counts and a cryptographic hash on-chain. Anyone can independently verify.

Who Does What

Admin
1. Create → 3. Finalize → 5. Open → 7. Close → 10. Commit
Committee
2. Register Keys → 8. Decrypt Shares
Voters
4. Register → 6. Vote

Key Properties

🕵

Anonymous — ZK proofs break the link between your identity and your vote. Nobody can tell who voted for what.

🔒

Encrypted — Votes are sealed until the committee decrypts. No one sees results early — not even the admin.

✅

Verifiable — All data is on-chain. Anyone can re-tally from the raw encrypted votes and verify the results match.

👥

Threshold trust — No single person can decrypt votes. A quorum of committee members must cooperate (e.g., 2 of 3).

Signup Gates — Who Can Vote?

🌐

Open

Anyone with the election link can sign up. No restrictions.

No wallet needed
🎫

Invite Codes

Admin generates one-time codes. Each code lets one voter sign up.

No wallet needed
📋

Allowlist

Admin provides a list of known identifiers (emails, names, IDs). Voters enter theirs to sign up.

No wallet needed
🔐

Admin Only

Only the admin can register voters directly by their identity commitment.

Wallet required
